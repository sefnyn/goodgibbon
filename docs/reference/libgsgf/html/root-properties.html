LIBGSGF Reference Manual

Root Properties

Root properties are only allowed in the first node of a GSGFGameTree.

AP (Application)

Type: GSGFComposed: (GSGFSimpleText:GSGFSimpleText)

The first value denotes the application, the second one the version. Both values are treated as opaque.

See http://www.red-bean.com/sgf/properties.html#AP.

CA (Charset)

Type: GSGFSimpleText

The charset of the SGF file. Note that this will always be the string "UTF-8" because all SGF files are converted to UTF-8 by this library.

See http://www.red-bean.com/sgf/properties.html#CA.

FF (File Format)

Type: GSGFNumber.

The number must be a positive number greater than zero. When writing, this number will always be 4. When reading, the number is ignored.

See http://www.red-bean.com/sgf/properties.html#FF.

GM (Game Type)

Type: GSGFNumber.

The number must be a positive number greater than zero. You can get a list of well-known game types from http://www.red-bean.com/sgf/properties.html#GM.

ST (Variation Style)

Type: GSGFNumber

Bit mask that determines how variations are shown. Valid values are in the range 0-3.

See http://www.red-bean.com/sgf/properties.html#ST.

SZ (Board size)

Type: GSGFNumber or GSGFComposed: (GSGFNumber:GSGFNumber)

Number of columns (first value) and rows (second value) of the board. Contrary to the SGF specification, the same value for columns and rows is allowed. In this case a single GSGFNumber is used instead of a GSGFComposed.

See http://www.red-bean.com/sgf/properties.html#SZ.
