The following properties can be used in all flavors of SGF. You should note that some of them are bound to abstract classes in libgsgf. When parsing SGF data, the values are cooked into the corresponding derived classes of the particular GSGFFlavor in use.

For example, for backgammon, whenever a GSGFMove is specified as the type, you will really get a GSGFMoveBackgammon. The same applies to GSGFPoint (cast to GSGFPointBackgammon, and to GSGFStone (cast to GSGFStoneBackgammon,

Move Properties

B (Black move)

Type: GSGFMove:

A move for player Black.

See http://www.red-bean.com/sgf/properties.html#B.

KO (Illegal move)

Type: GSGFEmpty:

Mark a move node as illegal. This property is a no-op for applications based on libgsgf.

See http://www.red-bean.com/sgf/properties.html#KO.

MN (Move number)

Type: GSGFNumber:

Set move number. This property is only interesting for incomplete games. Otherwise you should just rely on chronological move numbers.

See http://www.red-bean.com/sgf/properties.html#MN.

W (White move)

Type: GSGFMove:

A move for player White.

See http://www.red-bean.com/sgf/properties.html#W.
