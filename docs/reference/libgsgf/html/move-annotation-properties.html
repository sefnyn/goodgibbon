Move Annotation Properties

If libgsgf encounters one of the following four properties without a move in the same node (B or W the annotation property is silently discarded .

Having more than one move annotation property in one and the same node is an error.

BM (bad move)

Type: GSGFDouble

The played move is bad.

See http://www.red-bean.com/sgf/properties.html#BM.

DO (doubtful move)

Type: GSGFEmpty

The played move is doubtful.

See http://www.red-bean.com/sgf/properties.html#DO.

IT (interesting move)

Type: GSGFDouble

The played move is interesting.

See http://www.red-bean.com/sgf/properties.html#IT.

TE (good move or "tesuji")

Type: GSGFDouble

The played move is good. In Go this is called a "tesuji".

See http://www.red-bean.com/sgf/properties.html#TE.
